# Gravii User Landing

Gravii User Landing is the public Gravii landing and partner acquisition surface. It is built with Next.js 16, React 19, TypeScript, Bun, CSS Modules, GSAP, Lenis, and an app-local WebGPU hero renderer with a WebGL fallback.

## Scope

- `/` renders the public landing page.
- `/partners` renders the partner acquisition page and hands off to `partner.gravii.io`.
- `/privacy` and `/terms` render public legal pages.
- `/robots.txt`, `/sitemap.xml`, and `/manifest.webmanifest` are generated by Next metadata routes.
- Waitlist submission is handled through `app/(site)/actions.ts` and the Landing API.

## Stack

## Waitlist Behavior

The live form uses the server action in `app/(site)/actions.ts`.

Current behavior:

- Calls `POST /api/v1/landing/waitlist` through the Landing API base URL
- Forwards `?ref=GRV-XXXXXX` as `referral_code`
- Keeps honeypot and rate limit protections
- Stores successful submission state in `localStorage` under `gravii_waitlist`
- Shows returning visitors their existing referral code instead of resubmitting

## Troubleshooting

### Port 3000 is already in use

Another local process is listening on port 3000. Start Next on another port:

```bash
bunx next dev --port 3001
```

### Waitlist submission returns 403 in production

The waitlist form is backed by a Next.js Server Action. Next.js rejects Server Action POSTs before app code runs when the request `Origin` does not match the forwarded host. Production domains are allowlisted in `next.config.ts`; if a new custom domain, proxy, or preview domain is added, update `experimental.serverActions.allowedOrigins` and redeploy.

### Styles look stale

Regenerate style config and restart the dev server:

```bash
bun run setup:styles
bun dev
```
